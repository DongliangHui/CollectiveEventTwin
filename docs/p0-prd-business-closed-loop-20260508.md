# CollectiveEventTwin P0 PRD：业务闭环验证版

日期：2026-05-08

状态：P0 PRD draft，基于已确认 TR0 方案 A

关联 TR0：`docs/p0-tr0-scope-20260508.md`

## 1. 产品定义

P0 是一个基于授权或测试数据的网络风险世界线闭环系统。

它将网络信号、人工上传材料或授权样例转为可复核的风险因子、证据链、主线、World State、世界线分支、Agent Council 研判、决策简报和处置任务，用于验证完整业务闭环是否成立。

P0 不追求全网实时数据能力，不接真实平台 connector 作为主验收路径，不替代事实调查、线下处置、官方决策或真实民意。

## 2. P0 目标

### 2.1 核心目标

验证以下业务链路可解释、可复核、可测试、可扩展：

```text
授权/测试数据
-> Signal
-> RiskFactor
-> Evidence
-> Mainline
-> WorldState
-> WorldlineNode
-> CouncilResult
-> Report
-> Task
-> Audit
```

### 2.2 用户可见目标

用户在 P0 中应能：

1. 从高敏网络信号进入事件研判。
2. 查看信号来源、可信度、风险因子、证据状态和传播态势。
3. 区分事实、传言、观点、情绪和传播结构。
4. 人工确认或修正风险因子。
5. 基于信号和证据生成主线草稿。
6. 确认主线并生成 World State。
7. 运行 24-72 小时世界线推演。
8. 对关键节点启动 Agent Council。
9. 将研判结果转成决策简报和处置任务。
10. 查看关键判断的来源、人工确认和审计记录。

## 3. 非目标

P0 明确不做：

- 真实 Douyin、Kuaishou、Weibo、WeChat、Xiaohongshu、论坛 connector。
- 全网快搜生产能力。
- 生产级实时流处理。
- 无人机、3D、视频流实时分析。
- 多租户、复杂组织权限、商业化配置中心。
- 全量 20+ 页面一次性交付。
- 自动发布回应、自动决策、线下指挥。
- 个人画像、个人追踪、个人处置建议。
- 将 Agent 输出作为事实来源、真实民意或最终裁决。

## 4. P0 用户与使用场景

### 4.1 值班监测员

目标：

- 发现当前最高风险信号。
- 判断是否需要升级为事件研判。
- 避免把热度、传言和事实混为一谈。

P0 中的关键动作：

- 查看高敏信号。
- 查看风险因子和证据摘要。
- 创建或进入事件闭环。
- 标记继续观察或进入主线建模。

### 4.2 风险研判员

目标：

- 组织证据。
- 确认风险因子。
- 构建主线。
- 判断是否可进入推演。

P0 中的关键动作：

- 查看信号详情。
- 调整事实/传言/观点状态。
- 确认或驳回因子。
- 带缺口确认主线。
- 生成 World State。

### 4.3 决策材料准备者

目标：

- 获取可解释的研判结论。
- 看清证据缺口和不确定性。
- 将研判转成任务、回应准备和后续监测。

P0 中的关键动作：

- 查看世界线分支。
- 启动 Agent Council 压力测试。
- 生成决策简报。
- 创建处置任务。

### 4.4 复核/审计角色

目标：

- 确认高风险判断是否有证据。
- 确认敏感信息是否脱敏。
- 确认报告是否经过人工确认。

P0 中的关键动作：

- 查看判断链路。
- 查看 Agent 输出依据。
- 查看人工确认记录。
- 查看导出和任务审计。

## 5. P0 黄金场景

### 5.1 主场景

`CASE-CAMPUS-001 / campus-death-high-intensity`

场景摘要：

校园坠亡与疑似欺凌引发线下聚集风险。系统需要识别死亡事实、家属聚集、短视频扩散、回应可信度缺口、未成年人隐私泄露和网暴风险。

核心问题：

- 哪些事实已确认，哪些仍是待核验线索？
- 哪些网络信号正在推动情绪和责任归因？
- 哪些风险因子需要立即触发任务？
- 哪条主线最可能进入未来 24-72 小时高风险路径？
- 如果回应仍然笼统，会怎样影响线下聚集和责任叙事？
- 决策简报需要给出哪些行动任务和不确定性提示？

### 5.2 泛化 smoke 场景

P0 必须增加一个非校园最小场景，用于验证系统不是校园模板硬编码。

建议 smoke 场景：

`CASE-COMMUNITY-WATER-001 / community-water-outage-trust-risk`

场景摘要：

社区停水维修公告解释不足，居民在公开社区论坛和短视频评论中质疑物业、街道和供水单位响应不透明，出现线下集体咨询和投诉组织迹象。

需要覆盖的因子：

- 民生服务中断。
- 质疑不透明。
- 责任归因到物业/街道/供水单位。
- 线下集体咨询。
- 官方回应不足。
- 证据缺口。

smoke 验收目标：

- 能生成不同于校园事件的风险因子组合。
- 能进入同一对象链路。
- 能生成主线、世界线、Agent Council 和简报任务。

## 6. 核心流程

### 6.1 入口到信号

用户从 P0 入口看到高敏信号或事件卡。

系统展示：

- 事件或信号标题。
- 来源类型。
- 时间。
- 区域。
- 热度或传播变化。
- 事实可信度。
- 成线风险。
- 风险因子初步识别。
- 是否需要人工确认。

输出：

- `Signal` 列表。
- 待确认 `RiskFactor`。
- 证据摘要。

### 6.2 信号到证据

用户查看信号详情，系统必须拆分：

- 已确认事实。
- 待核验事实。
- 传言。
- 观点。
- 情绪表达。
- 传播结构。

用户可执行：

- 标记采信。
- 标记不采信。
- 降级为待核验。
- 标记含敏感信息。
- 加入主线草稿。

输出：

- `Evidence` 状态更新。
- 人工处理记录。
- 审计记录。

### 6.3 证据到风险因子

系统根据证据和信号生成风险因子建议。

用户必须能：

- 确认因子。
- 驳回因子。
- 调整因子可信度。
- 添加遗漏因子。
- 查看因子触发原因。

输出：

- 已确认或待确认 `RiskFactor`。
- 由因子触发的建议任务。

### 6.4 风险因子到主线

系统根据已选信号、证据和因子生成主线草稿。

主线必须展示：

- 起点信号。
- 关键诱因。
- 支点。
- 证据缺口。
- 涉及主体。
- 进入推演前检查。

用户可执行：

- 确认主线。
- 带缺口确认。
- 返回补证。
- 排除无关信号。

输出：

- `Mainline`。
- `WorldState` 输入包。

### 6.5 主线到世界线

系统基于 World State 生成 24-72 小时世界线。

世界线必须展示：

- 分支节点。
- 节点风险。
- 分支概率。
- 支点变化。
- 关键假设。
- 证据缺口。
- 是否建议 Agent Council。

输出：

- `WorldlineNode` 列表。
- 关键节点上下文。

### 6.6 世界线到 Agent Council

用户对关键节点启动 Agent Council。

Agent 只能读取：

- World State。
- 当前节点。
- 已确认或待核验证据。
- 风险因子。
- 支点状态。
- 用户输入的假设。

Agent 必须输出：

- 角色。
- 立场。
- 对假设的反应。
- 影响的支点。
- 分支概率变化。
- 依据。
- 不确定性。
- 不允许进入结论的内容。

输出：

- `CouncilResult`。
- 世界线回写建议。

### 6.7 Agent Council 到简报和任务

系统生成决策简报。

简报必须包含：

- 当前判断。
- 证据链摘要。
- 主要风险路径。
- Agent Council 关键变化。
- 不确定性和证据缺口。
- 建议动作。
- 任务清单。
- 后续监测点。
- 合规提示。

用户可执行：

- 标记结论已复核。
- 创建任务。
- 更新任务状态。
- 导出或保存简报。

输出：

- `Report`。
- `Task`。
- `AuditLog`。

## 7. P0 功能需求

### 7.1 数据与来源策略

P0 必须支持以下来源模式：

- `test_fixture`
- `manual_upload`
- `authorized_export`
- `official_api`
- `public_web`
- `third_party_contract`

P0 主验收只使用 `test_fixture`、`manual_upload` 或授权样例。

每条来源必须记录：

- 来源 ID。
- 来源类型。
- 授权模式。
- 原始链接或导入说明。
- 采集/导入时间。
- 可信度。
- 保留策略。
- 是否包含敏感信息。

### 7.2 信号检索与详情

P0 必须支持：

- 按案例查看信号。
- 按来源、时间、区域、因子、状态筛选。
- 查看信号摘要、来源、可信度、标签、相关证据。
- 将信号加入主线草稿。
- 排除无关信号并记录原因。

### 7.3 证据链

P0 必须支持：

- 事实、待核验、传言、观点、情绪、传播材料的状态区分。
- 证据可信度展示。
- 证据来源引用。
- 采信/不采信/待核验状态更新。
- 含敏感信息默认脱敏。
- 证据状态变更审计。

### 7.4 风险因子

P0 必须支持高价值因子 v0：

- 死亡/重伤。
- 未成年人。
- 家属到场。
- 现场视频。
- 质疑隐瞒。
- 质疑不作为。
- 线下聚集。
- 官方/机构回应。
- 隐私曝光。
- 网暴。
- 证据缺口。
- 跨平台传播。
- 民生服务中断。
- 集体投诉/咨询。

每个因子必须展示：

- 触发原因。
- 支撑证据。
- 可信度。
- 状态。
- 影响的页面模块或任务。

### 7.5 主线建模

P0 必须支持：

- 从信号草稿生成主线。
- 展示主线结构。
- 展示诱因、支点、证据缺口和涉及主体。
- 人工确认主线。
- 带缺口确认。
- 生成 World State 输入包。

### 7.6 世界线推演

P0 必须支持：

- 基于 World State 生成有限世界线。
- 展示 24-72 小时内关键节点。
- 展示分支概率、风险和支点变化。
- 对关键节点启动 Agent Council。
- 接收 CouncilResult 后更新节点解释或概率变化。

### 7.7 Agent Council

P0 必须支持：

- 基于世界线节点创建 Council Session。
- 使用固定 Agent 编队。
- 结构化输出反应、支点 delta、分支概率 delta。
- 标记依据和不确定性。
- 拦截无证据事实断言。
- 将结果回写世界线或简报。

校园黄金场景默认 Agent：

- 家属与亲属共同体。
- 校方。
- 学生群体。
- 教育主管与属地部门。
- 公众与媒体/KOL。

非校园 smoke 场景默认 Agent：

- 居民。
- 物业或服务单位。
- 街道/属地。
- 行业主管或公共服务单位。
- 本地公众/媒体。

### 7.8 决策简报和任务

P0 必须支持：

- 生成结构化简报。
- 展示主要判断、证据、缺口、世界线路径、Agent Council 结果。
- 生成处置任务。
- 更新任务状态。
- 记录任务来源和回写位置。

任务类型至少包括：

- 事实核验。
- 证据保全。
- 沟通机制。
- 响应准备。
- 隐私保护。
- 后续监测。

### 7.9 审计和人工确认

P0 必须支持：

- 高风险判断人工确认状态。
- 因子确认/驳回记录。
- 证据状态变更记录。
- Agent Council 输出记录。
- 简报生成和导出记录。
- 任务创建和状态更新记录。

未经人工确认的高风险判断不得进入正式简报结论区。

## 9. 验收标准

### 9.1 黄金路径验收

1. 用户能从校园黄金场景入口完成完整闭环。
2. 用户能查看 `SIG-001 / SIG-012 / SIG-017 / SIG-024` 的来源、可信度和证据。
3. 用户能确认至少 5 个风险因子。
4. 用户能生成并确认 `ML-001`。
5. 用户能生成 `WS-001`。
6. 用户能查看世界线节点，并聚焦 `NODE-C3`。
7. 用户能启动 `COUNCIL-001`。
8. CouncilResult 能展示支点变化和分支概率变化。
9. 用户能生成 `REPORT-001`。
10. 用户能创建或更新至少 3 个任务。

### 9.2 泛化 smoke 验收

1. 非校园 fixture 能进入同一对象链路。
2. 系统能识别非校园因子组合。
3. 系统能生成主线和 World State。
4. 系统能运行世界线和 Agent Council。
5. 简报输出不出现校园场景硬编码。

### 9.3 合规和安全验收

1. 未成年人姓名、照片、班级、住址等敏感信息默认脱敏。
2. 被标记为未授权的来源不能进入处理链。
3. Agent 试图输出无证据事实时，结果必须被标记或拦截。
4. 未人工确认的高风险判断不能进入正式结论区。
5. 每次证据状态、因子确认、主线确认、简报生成和任务更新都有审计记录。

### 9.4 性能验收

| 能力 | P0 目标 |
| --- | ---: |
| 案例/信号加载 | < 3s |
| 信号详情加载 | < 3s |
| 相关信号推荐 | < 5s |
| 主线生成/加载 | < 3s |
| 世界线推演 | < 180s |
| Agent Council | < 180s |
| 简报生成 | < 90s |

## 10. 成功指标

P0 成功不以真实数据源数量衡量，而以业务闭环可用性衡量。

必须达成：

- 完整闭环一次通过。
- 黄金场景所有对象 ID 连续可追踪。
- 非校园 smoke 场景证明泛化。
- 高风险判断 100% 带证据引用或不确定性标记。
- 敏感信息默认脱敏。
- Agent 输出 100% 符合结构化 schema。
- 简报和任务能追溯到证据、因子或世界线节点。

## 11. 风险与处理

| 风险 | P0 处理 |
| --- | --- |
| P0 被扩大成真实数据平台 | 明确 connector 不在主验收路径。 |
| 静态页面变成生产包袱 | 只把当前静态页作为行为契约。 |
| 单案例硬编码 | 增加非校园 smoke fixture。 |
| Agent 编造事实 | 结构化输入、输出校验、证据边界和人工确认。 |
| 热度误作事实 | 事实/情绪/传播三分离。 |
| 隐私信息泄露 | 默认脱敏、最小展示、审计。 |
| 验收不可重复 | 使用 golden fixture、smoke fixture、negative fixture。 |
