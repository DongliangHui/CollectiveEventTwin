## 1. 当前核心结论

在线下数据难以稳定获取的情况下，系统的主逻辑需要从“线下事实数据驱动”调整为“网络热度与舆论结构驱动”。

系统不应把目标定义为“还原完整事实真相”，而应定义为：

> 基于短视频、直播、微博、评论、新闻、论坛、搜索热度等网络信号，识别公众正在如何理解事件、情绪正在往哪里走、哪些叙事正在成形，并据此推演事件后续可能走向。

也就是说，系统判断的核心对象从“事件事实本身”扩展为：

```text
事件事实线索
+ 网络热度
+ 评论风向
+ 叙事框架
+ 责任归因
+ 传播结构
+ 响应动作
= 未来走向概率
```

这更适合当前阶段的数据可得性，也更容易形成业务闭环。

## 2. 新系统定位

新系统不是普通舆情监测，也不是简单情绪分析。

它应被定义为：

> 基于网络事件热度、舆论结构和多主体反应的风险世界线推演系统。

系统回答的问题不是：

```text
这件事真实情况到底是什么？
```

而是：

```text
公众正在如何理解这件事？
哪个叙事正在成为主线？
哪些情绪和归因正在推动升温？
哪些响应动作可能改变走向？
未来 24-72 小时可能分裂成哪些路径？
```

## 3. 完整业务闭环

推荐业务闭环如下：

```mermaid
flowchart LR
  A["网络内容接入"] --> B["热度与舆论风向识别"]
  B --> C["话题聚类与事件识别"]
  C --> D["叙事主线建模"]
  D --> E["世界状态生成"]
  E --> F["世界线推演"]
  F --> G["Agent Council 多主体反应校准"]
  G --> H["响应策略沙盘"]
  H --> I["研判报告与行动任务"]
  I --> J["舆情反馈回流"]
  J --> B
```

这个闭环的关键是：

```text
不等待线下事实完整。
用网络热度和舆论结构持续校准判断。
把事实缺口转化为验证任务和监测指标。
```

## 4. 核心判断变量

系统需要围绕 7 类核心变量建模。

### 4.1 热度强度

用于判断事件是否正在进入高关注状态。

可计算指标：

- 发帖量
- 转发量
- 评论量
- 短视频播放量
- 直播观看量
- 搜索指数
- 热度增长速度
- 热度加速度
- 跨平台热度迁移速度

关键判断：

```text
热度高不等于风险高。
热度快速上升且伴随情绪集中，才是风险进入推演的信号。
```

### 4.2 情绪方向

用于判断舆论正在升温、疲劳、分裂还是缓和。

应识别的情绪：

- 愤怒
- 恐惧
- 同情
- 质疑
- 嘲讽
- 支持
- 无力感
- 观望
- 对立情绪

关键判断：

```text
愤怒和责任归因结合，容易推动升级。
质疑和证据不清结合，容易拉长争议。
疲劳和新证据缺失结合，容易自然降温。
```

### 4.3 叙事框架

用于判断公众正在把事件理解成什么问题。

常见叙事框架：

- 公平问题
- 安全问题
- 责任问题
- 资源分配问题
- 群体对立问题
- 官方回应问题
- 规则透明问题
- 外部势力或阴谋叙事
- 人道或道德叙事

关键判断：

```text
叙事框架决定事件后续如何被传播。
同一事实，在不同叙事框架下会走向不同世界线。
```

### 4.4 责任归因

用于判断矛盾是否开始集中到某个主体。

应识别：

- 归因对象是谁
- 归因是否集中
- 归因是否从事件本身转向制度或组织
- 是否出现“必须回应”的主体
- 是否出现替罪羊或误伤对象

关键判断：

```text
归因越集中，主线越容易成形。
归因越分散，舆论更可能分裂或拉长。
```

### 4.5 证据强度

用于判断传播冲击力和可信度之间的关系。

证据类型：

- 原始视频
- 现场直播
- 现场图片
- 评论截图
- 长文爆料
- 媒体报道
- 官方回应
- 第三方机构数据
- 历史相似案例

关键判断：

```text
视频和直播传播冲击力强，但不天然等于事实可信。
系统需要同时展示传播强度和来源可信度。
```

### 4.6 扩散结构

用于判断事件是局部热、平台热，还是跨圈层扩散。

应识别：

- 单平台爆发
- 跨平台迁移
- KOL 带动
- 媒体放大
- 普通用户自发扩散
- 社群内循环
- 异常账号集中转发
- 二创扩散

关键判断：

```text
跨平台迁移和二创扩散，通常意味着事件脱离原始语境，进入新的叙事竞争。
```

### 4.7 行动转化风险

用于判断网络事件是否可能向线下或现实行动转化。

应识别：

- 线下聚集讨论
- 集体投诉
- 联名行动
- 抵制或拒绝配合
- 打卡围观
- 跨地区模仿
- 有组织话术
- 明确行动号召

关键判断：

```text
行动转化风险不只看热度，还要看组织性、地点性、时间窗口和号召强度。
```

## 5. 大模型与算法能力分工

完整版本应充分使用大模型和算法能力，但两者职责要分清。

### 5.1 算法负责可量化判断

适合算法完成：

- 热度趋势预测
- 情绪分类
- 话题聚类
- 评论立场分类
- 异常传播识别
- KOL 和社群传播图谱
- 跨平台迁移识别
- 相似历史案例匹配
- 时间序列预测
- 分支概率更新
- 触发阈值监测

算法输出应包括：

```text
指标值
变化趋势
异常检测
置信区间
触发原因
```

### 5.2 大模型负责语义解释和结构化推理

适合大模型完成：

- 评论核心诉求抽取
- 叙事框架识别
- 责任归因总结
- 群体观点摘要
- 视频/直播文本摘要
- 多源证据解释
- 为什么某条信号值得进入主线
- 为什么某个分支概率上升
- Agent Council 中多主体反应模拟
- 响应策略影响解释
- 研判报告生成

大模型输出不应直接作为最终判断，应带有：

```text
依据
不确定性
证据缺口
人工确认项
```

## 6. 新对象流转模型

建议统一对象层级如下：

```text
Online Content 网络内容
→ Data Point 数据点
→ Opinion Signal 舆论信号
→ Topic Cluster 话题聚类
→ Narrative Frame 叙事框架
→ Support Point 支点
→ Mainline 舆论主线
→ Opinion World State 舆论世界状态
→ Worldline Node 世界线节点
→ Council Result 多主体研判结果
→ Action Task 响应任务
→ Feedback Signal 反馈信号
→ Case Memory 案例沉淀
```

每个对象都需要保留：

- 来源
- 时间
- 平台
- 热度指标
- 情绪指标
- 叙事标签
- 责任归因
- 可信度
- 证据缺口
- 人工确认状态

## 7. 页面体系调整建议

当前封存版本可以作为视觉与交互基线，但下一版页面主题需要围绕网络热度和舆论结构重排。

### 7.1 驾驶舱

驾驶舱关注：

- 当前最热事件
- 热度上升最快事件
- 情绪恶化事件
- 跨平台扩散事件
- 高风险短视频/直播
- 评论归因变化
- 可能线下转化信号
- 需要人工确认的高影响信号

驾驶舱不应只列风险事件，而应展示：

```text
事件热度
情绪走势
叙事变化
传播结构
响应窗口
```

### 7.2 数据 / 信号检索页

数据页需要强化：

- 平台来源
- 内容类型
- 视频/直播/评论/转发关系
- 热度指标
- 评论风向
- 叙事标签
- 传播节点
- 是否进入主线草稿
- 为什么推荐相似数据

页面输出仍然是：

```text
当前主线草稿包
```

但草稿包的内容应从“证据材料”扩展为：

```text
热度证据
情绪证据
叙事证据
扩散证据
回应证据
缺口证据
```

### 7.3 舆论结构分析页

建议新增或合并为主线建模前置状态。

核心内容：

- 评论风向分布
- 主流叙事框架
- 责任归因对象
- 群体观点差异
- KOL/媒体/普通用户传播关系
- 平台间迁移路径
- 异常传播提示

### 7.4 主线建模页

主线不再只由事件证据组成，而应由舆论结构组成。

结构建议：

```text
起点内容
→ 热度爆点
→ 情绪诱因
→ 叙事支点
→ 责任归因
→ 传播放大点
→ 证据缺口
→ 人工确认主线
```

### 7.5 世界线推演页

世界线节点应表达：

- 当前舆论状态
- 叙事分裂点
- 情绪触发点
- 传播加速点
- 响应介入点
- 后续分支概率

典型分支：

- A 线：短期缓和
- B 线：低烈度持续
- C 线：二次升温
- D 线：跨平台外溢
- E 线：线下转化风险

### 7.6 Agent Council

Agent 角色需要从“机构主体”扩展为“舆论生态主体”。

建议 Agent：

- 当事群体
- 普通围观者
- 短视频平台
- 微博/KOL
- 媒体编辑
- 官方回应部门
- 利益相关方
- 质疑群体
- 平台治理方

Agent Council 应回答：

```text
如果采取某种回应，哪些主体会接受？
哪些主体会质疑？
哪些叙事会被强化？
哪些分支概率会改变？
```

### 7.7 响应策略沙盘

建议作为完整版本的重要模块。

可模拟策略：

- 沉默观察
- 模糊回应
- 事实澄清
- 公开证据
- 道歉
- 第三方介入
- 平台辟谣
- 线下沟通
- 定向解释
- 分阶段回应

每种策略输出：

- 对热度的影响
- 对情绪的影响
- 对叙事的影响
- 对责任归因的影响
- 对线下转化风险的影响
- 可能副作用

### 7.8 汇报与任务闭环

汇报页输出不应只是结论，而应转化为行动包。

输出内容：

- 当前网络热度判断
- 主要叙事
- 情绪变化
- 升温原因
- 降温条件
- 关键风险路径
- 建议响应动作
- 监测指标
- 触发阈值
- 人工确认项
- 反馈回流计划

## 8. 新业务闭环中的“线下数据缺失”处理

线下数据难获取不是系统失败，而是系统需要显式表达的不确定性。

系统应把缺失数据转化为：

```text
证据缺口
验证任务
替代指标
持续监测条件
概率置信度限制
```

示例表达：

```text
当前 C 线“二次升温”概率为 41%。
该判断主要由短视频播放加速、评论责任归因集中、保险风险讨论升温共同推动。
由于缺少线下实际通航调度和原始保险报价，置信度限制在 0.72。
若 6 小时内仍无可信回应，且负面评论占比继续超过 58%，C 线概率将继续上调。
```

## 10. 风险与边界

系统必须明确边界：

- 不声称掌握完整事实真相。
- 不把网络热度等同于现实风险。
- 不把大模型输出当作最终裁决。
- 不做舆论操控建议。
- 不输出煽动性行动方案。
- 高风险判断必须保留人工确认。

系统可信度来自：

```text
证据链
解释链
不确定性
人工确认
反馈重算
```

## 11. 下一步建议

建议下一版从当前封存版本出发，按以下顺序推进：

1. 先重构对象模型：从事件风险对象扩展为网络内容、舆论信号、叙事框架、传播结构。
2. 重做驾驶舱：突出热度、情绪、叙事、扩散、响应窗口。
3. 重做数据页：加入平台来源、视频/直播/评论、热度指标和推荐理由。
4. 增加舆论结构分析状态：评论风向、责任归因、叙事框架、传播图谱。
5. 调整主线建模：由“证据链成线”升级为“舆论叙事成线”。
6. 调整世界线：分支概率由热度、情绪、叙事和响应动作共同驱动。
7. 扩展 Agent Council：加入平台、KOL、普通围观者、官方回应部门等主体。
8. 增加响应策略沙盘：测试不同回应策略对分支概率的影响。
9. 汇报页改为行动闭环：建议动作、监测指标、触发阈值、反馈回流。

## 12. 一句话总结

下一版系统的核心不是“监测事件”，而是：

> 监测网络社会如何理解事件，并用热度、情绪、叙事、传播结构和响应动作来推演未来走向。

在这个逻辑下，即使线下数据不足，系统仍然可以形成完整业务闭环：

```text
网络信号发现
→ 舆论结构识别
→ 主线建模
→ 世界线推演
→ 多主体反应校准
→ 响应策略测试
→ 报告与任务
→ 舆情反馈回流
→ 重算世界线
```
